# PRD.md - 반도체 시료 생산주문관리 시스템 기능 명세서

## 1. 프로젝트 개요

### 배경
가상의 반도체 회사 "S-Semi"는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품한다. 주문량 급증으로 인해 엑셀/메모장 기반 관리의 한계가 발생하여 체계적인 시료 생산주문관리 시스템을 개발한다.

### 시스템 특성
- 콘솔(터미널) 기반으로 동작
- 담당자가 직접 명령을 입력하여 시료 등록 및 주문 처리

---

## 2. 역할 구조

| 역할 | 설명 |
|------|------|
| 고객 | 시료 요청자. 필요한 시료를 이메일로 요청 |
| 주문 담당자 | 요청에 맞게 주문서를 작성하고 관리 |
| 생산 담당자 | 개발 시료 등록, 주문 수신 후 승인 또는 거절 |

---

## 3. 주문 상태 흐름

```
RESERVED → (승인) → 재고 충분  → CONFIRMED → RELEASE
                 → 재고 부족  → PRODUCING → CONFIRMED → RELEASE
         → (거절) → REJECTED
```

| 상태 | 의미 |
|------|------|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (모니터링 제외) |
| PRODUCING | 주문 승인 완료 + 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 + 출고 대기 중 |
| RELEASE | 출고 완료 |

---

## 4. 메인 메뉴

기능별 선택 화면을 표시하며, 전체 시료에 대한 요약 정보를 함께 확인할 수 있다.

| 메뉴 | 설명 |
|------|------|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 시료 주문 | 고객 주문 접수 |
| 주문 승인/거절 | 생산 라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 생산라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |
| 출고 처리 | CONFIRMED 상태 주문에 대해 출고 실행 |

## 5. 기능 상세 명세

### 5.1 시료 관리

시료(Sample)는 시스템의 가장 기본 단위이며, 시스템에 등록된 시료만 주문 가능하다.

#### 5.1.1 시료 등록
- 새로운 시료를 시스템에 추가한다.
- 입력 속성:
  - 시료 ID (고유값)
  - 시료 이름
  - 평균 생산시간 (min/ea, 양수 > 0)
  - 수율 (0 이상 1 이하 소수, 정상 시료 수 / 총 생산 시료 수), 0.0 <= yield_rate <= 1.0
    - 예) 100개 생산 중 정상 90개 = 0.9
  - 초기 재고 수량 (사용자가 직접 입력, 0 이상 정수)

#### 5.1.2 시료 목록 조회
- 등록된 모든 시료 목록을 출력한다.
- 시료 ID, 이름, 평균 생산시간, 수율, 현재 재고 수량을 함께 표시한다.

#### 5.1.3 시료 검색
- 이름 등 속성으로 특정 시료를 검색한다.

---

### 5.2 시료 주문

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

#### 5.2.1 시료 예약 (주문 접수)
- 입력 값:
  - 시료 ID
  - 고객명
  - 주문 수량
- 접수 시 주문 상태는 `RESERVED`로 설정된다.
- 주문번호는 자동 생성된다. (예: `ORD-YYYYMMDD-XXXX`)

---

### 5.3 주문 승인/거절

접수된 주문(RESERVED) 목록을 확인하고 특정 주문에 대해 승인 또는 거절 처리한다.

#### 5.3.1 접수된 주문 목록
- RESERVED 상태의 주문 목록을 표시한다.

#### 5.3.2 주문 승인
- 재고 상황에 따라 자동으로 두 가지 방식으로 처리된다.
  - **재고 충분** → 즉시 `CONFIRMED` 상태로 전환
  - **재고 부족** → 생산 라인에 자동 등록, 주문 상태를 `PRODUCING`으로 전환

#### 5.3.3 주문 거절
- 즉시 `REJECTED` 상태로 전환한다.

---

### 5.4 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 구성한다.

#### 5.4.1 주문량 확인
- 현재 상태별 주문 수를 표시한다.
  - 대상 상태: RESERVED, CONFIRMED, PRODUCING, RELEASE
  - REJECTED는 유효한 주문이 아니므로 제외한다.

#### 5.4.2 재고량 확인
- 각 시료별 현재 재고 수량을 표시한다.
- 주문 대비 재고 수량에 따라 상태를 표기한다.

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량이 0 |

---

### 5.5 생산라인

주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산한다.

#### 5.5.1 생산량 계산
- **실 생산량**: `ceil(부족분 / (수율 × 0.9))`
- **총 생산 시간**: `평균 생산시간 × 실 생산량`

#### 5.5.2 생산 완료 처리
- 생산 완료 시 주문 상태를 `PRODUCING` → `CONFIRMED`으로 변경한다.

#### 5.5.3 생산 현황 표기
- 현재 생산 중인 시료에 대한 정보를 표시한다.
  - 주문 정보, 현재까지의 생산량 등

#### 5.5.4 대기 주문 확인
- 생산 큐에서 대기 중인 주문 목록을 출력한다.
- 스케줄링 전략: **FIFO** (선입선출)

---

### 5.6 출고 처리

재고가 충분해진 CONFIRMED 주문에 대해 출고를 처리한다.

- CONFIRMED 상태의 주문 목록을 표시한다.
- 특정 주문을 선택하여 출고를 실행한다.
- 출고 처리 시 주문 상태가 `RELEASE`로 전환된다.

---

## 6. 생산 라인 정의

- 공장에서 시료 하나를 생산하는 설비 흐름
- 하나의 생산 라인은 시료를 **하나씩** 생산
- **주문이 들어온 시료에 대해서만** 생산

---

## 7. 제약 사항

- 시스템에 등록되지 않은 시료는 주문 불가
- REJECTED 주문은 모니터링 대상에서 제외
- 생산 라인 스케줄링은 FIFO 방식만 사용
- 명시되지 않은 기능은 구현하지 않음
